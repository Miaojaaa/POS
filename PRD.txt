## 1. ภาพรวมของระบบ

### 1.1 วัตถุประสงค์
พัฒนาระบบบริหารจัดการร้านเสริมสวยแบบครบวงจร ที่รองรับการทำงานตั้งแต่การรับลูกค้าที่เคาน์เตอร์ (POS) ไปจนถึงการบริหารทรัพยากรบุคคล (HR) และการวิเคราะห์ธุรกิจ (Dashboard) ในแพลตฟอร์มเดียว

### 1.2 ขอบเขตของระบบ

| โมดูล | คำอธิบาย |
|-------|----------|
| POS | ระบบขายหน้าร้าน รับออร์เดอร์ คิวลูกค้า และชำระเงิน |
| CRM | ระบบสมาชิก คูปอง Wallet และประวัติลูกค้า |
| ERP | ระบบสต็อกสินค้าและวัตถุดิบเคมี (ระดับมิลลิกรัม) |
| Dashboard | รายงานยอดขาย ต้นทุน กำไร และค่าใช้จ่าย |
| HR & Payroll | ระบบค่าคอมมิชชั่น Pools KPI และเงินเดือน |

### 1.3 ผู้มีส่วนได้ส่วนเสียหลัก
- **Owner / เจ้าของร้าน** — เข้าถึงได้ทุกโมดูล รวมถึง Dashboard กำไรจริง
- **Manager / ผู้จัดการ** — อนุมัติส่วนลด อนุมัติการเบิกสต็อก และ Submit เงินเดือน
- **Cashier / พนักงานแคชเชียร์** — รับออร์เดอร์และชำระเงิน
- **ช่างและผู้ช่วยช่าง** — เข้าถึง POS เพื่อดูคิวและบันทึกการทำงาน

## 2. ผู้ใช้งานและบทบาท

### 2.1 ตารางสิทธิ์การเข้าถึง

| ฟีเจอร์ | Owner | Manager | Cashier | ช่าง/ผู้ช่วย |
|---------|:-----:|:-------:|:-------:|:------------:|
| เข้าถึง POS | ✅ | ✅ | ✅ | ✅ |
| ลดราคา / แก้ไขราคา | ✅ | ✅ (ต้องใช้ Key) | ❌ | ❌ |
| ดู Dashboard กำไร | ✅ (Owner Key) | ❌ | ❌ | ❌ |
| Submit เงินเดือน | ✅ | ✅ | ❌ | ❌ |
| อนุมัติโอนสต็อก | ✅ | ✅ | ❌ | ❌ |
| จัดการ CRM | ✅ | ✅ | ✅ | ❌ |
| เพิ่ม Ticket / เติม Wallet | ✅ | ✅ | ❌ | ❌ |

### 2.2 ระบบ Key / PIN

- **Manager Key** — ใช้สำหรับ: อนุมัติส่วนลด, แก้ไขราคา, อนุมัติการโอนสต็อก
- **Owner Key** — ใช้สำหรับ: เข้าถึงหน้า Dashboard กำไรจริง, ฟังก์ชันพิเศษระดับเจ้าของ
- Key จะเป็นรูปแบบ PIN 4–6 หลัก หรือ OTP ที่ตั้งค่าในระบบ

## 3. โมดูล POS

### 3.1 Workflow หลัก

```
ลูกค้าเข้าร้าน
      ↓
Consult (กรอกข้อมูล)
  - ชื่อลูกค้า / เบอร์โทร
  - ช่างและผู้ช่วยที่ดูแล
  - บริการที่ต้องการ
  - เคมีที่จะใช้ (ถาดสี / สูตร)
      ↓
ส่งเข้าคิว (Queue)
  - แสดงสถานะ: กำลังทำ / รอคิว
  - แสดงผู้รับผิดชอบ (ช่าง)
  - แสดงบริการที่ทำ
  - แสดงราคาโดยประมาณ
  - แก้ไขได้ระหว่างให้บริการ
      ↓
เช็คเอาท์ที่ Cashier
  - ตรวจสอบรายการ
  - ใส่ส่วนลด (ต้องใช้ Manager Key)
  - เลือกวิธีชำระ (เงินสด / โอน / Wallet)
  - พิมพ์ / ส่งใบเสร็จ
```

### 3.2 หน้า Consult (กรอกข้อมูลออร์เดอร์)

**ข้อมูลที่ต้องกรอก:**

| ฟิลด์ | ประเภท | หมายเหตุ |
|-------|--------|----------|
| ชื่อลูกค้า | Text / Dropdown (ค้นหาสมาชิก) | ค้นหาด้วยชื่อหรือเบอร์โทร |
| เบอร์โทรศัพท์ | Text | ถ้าเป็นสมาชิก ให้แสดง Modal ข้อมูลสมาชิก |
| ช่างผู้รับผิดชอบ | Dropdown | เลือกจากรายชื่อพนักงานที่ Active |
| ผู้ช่วยช่าง | Dropdown (Multi-select) | ไม่บังคับ |
| บริการ | Dropdown / Search | เลือกได้หลายรายการ |
| เคมีที่ใช้ (ถาดสี) | Dropdown จาก Stock | ระบุชื่อผลิตภัณฑ์ + ปริมาณ (มล./กรัม) |
| หมายเหตุ | Textarea | ข้อมูลเพิ่มเติม |

### 3.3 Modal แจ้งสมาชิก (เมื่อกรอกเบอร์โทรสมาชิก)

เมื่อพบข้อมูลสมาชิกในระบบ ให้แสดง Modal พร้อม:
- **ยอดคงเหลือใน Wallet**
- **Ticket / Coupon ที่มีอยู่**
- **ประวัติการแพ้สารเคมี** (สีแดง / เด่นชัด)
- **ประวัติการรับบริการ 5 ครั้งล่าสุด** (วันที่, บริการ, ช่าง, สารเคมีที่ใช้)

### 3.4 ระบบคิว (Queue Board)

แสดงข้อมูลแบบ Kanban หรือ List:
- **กำลังทำ** — ชื่อลูกค้า | ช่าง | บริการ | เวลาเริ่ม | เวลาโดยประมาณ
- **รอคิว** — ลำดับการรอ
- สามารถ **แก้ไข / ยกเลิก** ออร์เดอร์ที่อยู่ในคิวได้

### 3.5 การแก้ไขราคาและส่วนลด

- แก้ไขราคาได้เฉพาะเมื่อ Manager / Owner ยืนยัน Key
- ส่วนลดบันทึกเป็น % หรือจำนวนเงิน
- ระบบบันทึก log ว่าใครเป็นคนอนุมัติ ณ เวลาใด

### 3.6 ระบบต้นทุนเคมี (ถาดสี)

- เมื่อเลือกเคมีที่ใช้ในออร์เดอร์ ระบบจะดึงราคาต้นทุนต่อมิลลิกรัมจาก ERP
- คำนวณต้นทุนรวมต่อออร์เดอร์อัตโนมัติ
- ข้อมูลนี้ส่งต่อไปยัง Dashboard เพื่อคำนวณกำไรจริง

### 3.7 การชำระเงิน (Checkout)

ช่องทางชำระ:
- เงินสด
- โอนเงิน (QR Code / พร้อมเพย์)
- Wallet ของสมาชิก
- คูปอง / Ticket (ลดราคาบริการนั้น 100%)
- ผสมหลายช่องทาง

**กรณีใช้คูปอง:**
- ระบบจะลดราคาบริการที่ตรงกับ Ticket นั้น 100%
- ยอดส่วนนี้ **ไม่นับเข้า Pools ค่าคอม** แต่ **บันทึกประวัติการทำงานของช่าง** ไว้

## 4. โมดูล CRM

### 4.1 ระบบสมาชิก

**ข้อมูลสมาชิก:**

| ฟิลด์ | ประเภท | หมายเหตุ |
|-------|--------|----------|
| ชื่อ-นามสกุล | Text | |
| เบอร์โทรศัพท์ | Text (Unique) | ใช้เป็น Primary Key |
| วันเกิด | Date | |
| ประวัติการแพ้สารเคมี | Text (Permanent) | ไม่สามารถลบได้ — Permanent Record |
| Wallet Balance | Decimal | |
| ระดับสมาชิก | Enum | Basic / Silver / Gold |

### 4.2 ระบบ Wallet

- ลูกค้าซื้อ Course ยอดเงินถูกเติมเข้า Wallet ทันที
  - ตัวอย่าง: ซื้อ Course 30,000 บาท → Wallet = 30,000 บาท
- Manager / Owner เท่านั้นที่สามารถ **เติมเงิน** หรือ **ปรับยอด Wallet** ได้
- บันทึก Transaction ทุกรายการ (เติม / ใช้จ่าย / วันที่ / ผู้ดำเนินการ)

### 4.3 ระบบคูปอง / Ticket

**ประเภทของ Ticket:**
- Ticket บริการ (เช่น สปา AAA ฟรี 1 ครั้ง)
- Ticket ส่วนลด (เช่น ลด 20% สำหรับทำสี)
- Ticket สินค้า

**วิธีการทำงานของ Ticket:**
- เมื่อนำ Ticket มาใช้ใน Order ระบบจะ **ลดราคาบริการที่ตรงกัน 100%** ออกจาก Order โดยอัตโนมัติ
- ยอดที่ลด **ไม่นับเข้า Revenue Pools** ของช่าง แต่บันทึกเป็น **ประวัติชั่วโมงทำงาน** ของช่างคนนั้น
- Manager / Owner เท่านั้นที่สามารถออก Ticket ได้

### 4.4 ประวัติการรับบริการ

- เก็บประวัติย้อนหลัง **5 ครั้งล่าสุด** ต่อลูกค้า 1 คน
- ข้อมูลที่บันทึก: วันที่ | บริการ | ช่างที่ทำ | สารเคมีที่ใช้ | ยอดที่จ่าย
- ข้อมูลนี้แสดงใน Modal บน POS เมื่อค้นหาสมาชิก

### 4.5 ระบบ Line OA Integration

- ส่งข้อความแจ้งเตือนอัตโนมัติ เมื่อลูกค้าไม่ได้ใช้บริการเกินระยะเวลาที่กำหนด (ตั้งค่าได้ เช่น 30/60/90 วัน)
- ข้อความตัวอย่าง: "สวัสดีค่ะ คุณ [ชื่อ] ไม่ได้มาใช้บริการมา X วันแล้ว ฝากติดตามนะคะ 😊"
- การตั้งค่า: กำหนด Template ข้อความและ Trigger Period ได้

## 5. โมดูล ERP / ระบบสต็อก

### 5.1 โครงสร้างคลังสินค้า

ระบบมี **2 คลัง:**

| คลัง | ชื่อ | คำอธิบาย |
|------|------|----------|
| Main Warehouse | คลังหลัก | สินค้าที่ยังไม่ได้เปิด / สำรอง |
| Sub Warehouse | คลังหน้าร้าน | สินค้าที่เปิดใช้งานแล้ว / กำลังใช้อยู่ |

### 5.2 การกำหนดข้อมูลสินค้า

แต่ละรายการสินค้าต้องกำหนด:

| ฟิลด์ | ประเภท | ตัวอย่าง |
|-------|--------|---------|
| ชื่อสินค้า | Text | "ครีมเปลี่ยนสี X-Brand" |
| ปริมาณต่อขวด/หน่วย | Integer (มิลลิกรัม) | 500,000 มก. (= 500 กรัม) |
| จำนวนในสต็อก | Integer (ขวด/หน่วย) | 10 ขวด |
| จำนวนปริมาณที่เหลือในขวดปัจจุบัน | Integer (มิลลิกรัม) | Sub Warehouse เท่านั้น |
| ราคาต้นทุนต่อขวด | Decimal (บาท) | 850 บาท |
| ราคาต้นทุนต่อมิลลิกรัม | คำนวณอัตโนมัติ | = ราคาต่อขวด / ปริมาณ |
| จุดแจ้งเตือน (Reorder Point) | Integer (มก.) | กำหนดระดับขั้นต่ำ |

### 5.3 Logic การหักสต็อก

```
เมื่อ POS บันทึกการใช้เคมี:
  1. หักปริมาณ (มิลลิกรัม) จาก Sub Warehouse
  2. ถ้าปริมาณใน Sub Warehouse = 0 → หักจำนวนขวดออก 1
  3. ถ้าจำนวนขวดใน Sub Warehouse = 0 → แจ้งเตือน Manager ว่าสินค้าหมด
  4. ถ้าปริมาณรวมต่ำกว่า Reorder Point → แจ้งเตือนเพื่อสั่งซื้อ
```

### 5.4 การโอนสินค้าระหว่างคลัง

- การโอนจาก Main → Sub Warehouse ต้องได้รับการ **Approve จาก Manager / Owner**
- บันทึก Log: ใคร โอน อะไร กี่หน่วย เมื่อใด ใคร Approve

### 5.5 รายงานสต็อก

- สรุปสต็อกทั้งสองคลัง
- ประวัติการเบิก-จ่าย
- สินค้าใกล้หมด (ต่ำกว่า Reorder Point)
- มูลค่าสต็อกรวม

## 6. โมดูล Dashboard

> **หมายเหตุ:** การเข้าถึงหน้า Dashboard กำไรจริงต้องใช้ **Owner Key** เท่านั้น

### 6.1 ภาพรวมการใช้บริการ

- กราฟแสดงจำนวนลูกค้าแยกตามหมวดหมู่หลัก (ผม / เล็บ / สปา ฯลฯ)
- Drill-down ลงบริการย่อย (ตัดผม / ทำสี / สปาเท้า ฯลฯ)
- กรองตามช่วงเวลา (วัน / สัปดาห์ / เดือน / ปี)

### 6.2 รายงานรายได้และต้นทุน

| รายการ | คำอธิบาย |
|--------|----------|
| รายได้รวม | ยอดขายทั้งหมด (ไม่รวมออร์เดอร์ที่ใช้ Ticket ฟรี) |
| ต้นทุนเคมี | ดึงจากข้อมูลการใช้เคมีใน POS × ราคาต้นทุน |
| ค่าใช้จ่ายอื่น | Expense ที่กรอกเพิ่มเติม |
| กำไรสุทธิ | รายได้รวม - ต้นทุนเคมี - ค่าใช้จ่ายอื่น |

### 6.3 ระบบบันทึก Expense

Manager / Owner สามารถเพิ่มค่าใช้จ่ายพิเศษได้:

| ประเภท Expense | ตัวอย่าง |
|---------------|---------|
| ค่าสาธารณูปโภค | ค่าไฟ ค่าน้ำ |
| ค่าเช่า | ค่าเช่าพื้นที่ |
| วัสดุอุปกรณ์ | ถุงมือ ผ้าเช็ด ฯลฯ |
| เงินเดือนพนักงาน | บันทึกรายเดือน |
| ค่าใช้จ่ายอื่นๆ | ระบุเอง |

### 6.4 กราฟและรายงานที่แสดง

- Revenue vs. Cost (Bar Chart รายเดือน)
- กำไรสุทธิ (Line Chart)
- Top 5 บริการที่ทำรายได้สูงสุด
- ช่างที่สร้างรายได้สูงสุด

## 7. โมดูล HR & Payroll

### 7.1 โครงสร้างค่าคอมมิชชั่น (Pools System)

ค่าคอมมิชชั่นคำนวณจาก **Revenue หลังหักต้นทุนเคมี** แบ่งเป็น Pools ตามแผนก:

| แผนก | % ของ Revenue Pool |
|------|---------------------|
| ช่างใหญ่ | กำหนดได้ (ตัวอย่าง: 10%) |
| ผู้ช่วยช่าง | กำหนดได้ (ตัวอย่าง: 5%) |
| แผนกอื่นๆ | กำหนดได้ตามนโยบาย |

**สูตรคำนวณ:**
```
Pools แผนก = Revenue หลังหักต้นทุนเคมี × % แผนก
ค่าคอมต่อคน = Pools แผนก ÷ จำนวนพนักงานในแผนก
```

> ตัวอย่าง: Revenue 1,000,000 บาท → Pools ช่างใหญ่ (10%) = 100,000 บาท → มี 10 คน → คนละ 10,000 บาท

### 7.2 ค่าคอมมิชชั่นจากการขายสินค้า Retail

- ถ้าช่างขายสินค้า Retail หน้าร้านได้ → ได้ค่าคอม **20% ของราคาขาย**
- ค่าคอมนี้เป็น **Individual Commission** ไม่นับรวมกับ Pools

### 7.3 ระบบ KPI

ระบบจะคำนวณและแสดงตัวชี้วัดต่อไปนี้สำหรับช่างแต่ละคน:

| KPI | คำอธิบาย |
|-----|----------|
| จำนวนออร์เดอร์ที่รับ | นับ Orders ที่ assigned ในรอบเดือน |
| เปรียบเทียบกับค่าเฉลี่ยทีม | แสดงว่าสูง/ต่ำกว่า Average เท่าไหร่ |
| % การรับงาน | Orders ของช่างคนนี้ / Orders รวมทีม |
| รายได้ที่สร้าง | ยอดรวม Orders ที่ช่างคนนี้รับผิดชอบ |

**การแจ้งเตือน KPI:**
- ก่อนกด Submit เงินเดือน ระบบจะตรวจสอบว่ามีช่างใดที่ KPI ต่ำกว่าเกณฑ์ผิดปกติหรือไม่
- ถ้าพบ → แสดง Warning Modal พร้อมรายชื่อช่างที่น่าเป็นห่วง
- Manager ยังสามารถ Submit ต่อได้ แต่ต้องผ่านการยืนยัน

### 7.4 ประวัติการทำงาน (Work Log)

- บันทึกทุก Order ที่ช่างรับ รวมถึง Order ที่ใช้ Ticket ฟรี
- สรุปรายเดือน: จำนวน Orders | รายได้ | ค่าคอม | Retail Commission
- Export เป็น PDF / Excel ได้

## 9. Non-Functional Requirements

### 9.1 ความปลอดภัย

- ระบบ Authentication ด้วย Username + Password
- Manager Key / Owner Key แยกจาก Login ปกติ
- Log ทุก Action ที่ต้องใช้ Key พิเศษ (ใคร เมื่อไหร่ ทำอะไร)
- ข้อมูลประวัติการแพ้สารเคมีของลูกค้า → **ถาวร ลบไม่ได้**
- HTTPS ตลอดการสื่อสาร

### 9.2 Performance

- หน้า POS ต้องโหลดภายใน < 2 วินาที
- ระบบคิวต้อง Real-time (WebSocket หรือ Polling ทุก 10 วินาที)
- รองรับการทำงานพร้อมกันอย่างน้อย 5 Devices

### 9.3 Data Backup

- Backup ข้อมูลอัตโนมัติรายวัน
- สามารถ Export ข้อมูลสำคัญเป็น CSV / Excel / PDF

### 9.4 Audit Log

- บันทึกทุกการเปลี่ยนแปลงราคา, ส่วนลด, การโอนสต็อก, การออก Ticket
- ดูได้โดย Manager ขึ้นไป

## 10. Glossary

| คำศัพท์ | คำอธิบาย |
|---------|----------|
| POS | Point of Sale — ระบบขายหน้าร้าน |
| CRM | Customer Relationship Management — ระบบบริหารความสัมพันธ์ลูกค้า |
| ERP | Enterprise Resource Planning — ระบบบริหารทรัพยากร (ในที่นี้คือสต็อก) |
| Wallet | กระเป๋าเงินดิจิทัลของสมาชิก ใช้จ่ายค่าบริการในร้าน |
| Ticket | คูปองบริการที่ลูกค้าได้รับจากการซื้อ Course |
| Pools | ยอดรวมค่าคอมมิชชั่นระดับแผนก ก่อนหารตามจำนวนพนักงาน |
| Main Warehouse | คลังหลัก — สินค้าที่ยังไม่ได้เปิดใช้ |
| Sub Warehouse | คลังหน้าร้าน — สินค้าที่เปิดใช้งานแล้ว |
| KPI | Key Performance Indicator — ตัวชี้วัดผลงานพนักงาน |
| Manager Key | PIN พิเศษสำหรับ Manager ใช้อนุมัติส่วนลดและการเปลี่ยนแปลง |
| Owner Key | PIN พิเศษสำหรับ Owner เท่านั้น ใช้เข้าถึงข้อมูลกำไรจริง |
| Reorder Point | ระดับสต็อกขั้นต่ำที่ต้องสั่งซื้อเพิ่ม |
